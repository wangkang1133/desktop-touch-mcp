# v1.0 Milestone: Less Surface, More Meaning

The release of v1.0 marks a significant shift in how `desktop-touch-mcp` connects LLMs to the Windows desktop. It is not just a version bump; it is the culmination of the **Beyond Coordinate Roulette** philosophy.

---

## The Problem: Tool Fatigue and Blind Guessing

In earlier versions (v0.15 and prior), the MCP surface was cluttered with over 60 specialized tools. While powerful, this created two major problems:

1.  **Context Bloat:** LLMs had to process a massive catalog of tool descriptions, consuming valuable tokens.
2.  **Decision Friction:** Choosing between `mouse_click`, `click_element`, or `browser_click_element` forced the agent to reason about the *implementation* of the UI rather than its *meaning*.

This often resulted in "Coordinate Roulette"—the agent guessing a position and hoping for the best.

---

## The Solution: Semantic Condensation (LLM UX Improvement)

In v1.0, we have condensed the tool surface from **65 tools to 28**. This is a major **LLM UX improvement**: by reducing "Tool Fatigue" and context bloat, we provide a cleaner, more intuitive interface for AI agents. This leads to more robust decision-making, fewer hallucinations, and a significantly lower token overhead for every interaction.

### From Dispatchers to Meaning
Instead of dozens of top-level functions, v1.0 uses a few primary dispatchers. For example:
- `keyboard` handles all typing and key presses.
- `scroll` manages everything from raw mouse wheels to smart vertical list seeking.
- `browser_eval` absorbs DOM inspection and state extraction.

### Defaulting to the World-Graph
The most critical change is the promotion of **Anti-Fukuwarai v2 (World-Graph)** as the default interface. 
- **`desktop_discover`** allows the agent to see the desktop as a collection of semantic entities (buttons, inputs, windows) rather than a flat bitmap.
- **`desktop_act`** allows the agent to interact with those entities using temporary "leases" that are automatically guarded by the server.

By shifting to this model, we move from "clicking a coordinate" to "acting on an affordance."

---

## RPG in Action: Automated Trust

The **Reactive Perception Graph (RPG)** is now wired into the heart of the server. 
- **Auto-Perception:** The server now automatically monitors focus, identity, and modal obstructions. When an agent calls an action, the server verifies the "contract" between the agent's perception and the world's reality.
- **Lease Hardening:** Interaction rights (leases) are now payload-size aware and have TTLs that encourage the agent to refresh its view of the world proactively.

---

## Operating in the Background

With the introduction of **Background Input (WM_CHAR)**, the agent can now perform many tasks without constantly stealing the user's foreground focus. This makes the agent less intrusive and the interaction more robust against accidental user interference.

---

## Conclusion: A Safer Contract

The v1.0 milestone is about making the desktop a predictable, semantic world for LLMs. By reducing the surface and hardening the perception contract, we enable agents to act with confidence rather than hope.
